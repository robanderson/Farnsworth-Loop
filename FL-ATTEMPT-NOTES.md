# FL attempt notes

Fixed a fail-OPEN correctness bug in all five provider runner scripts (bin/{glm,local,minimax,codex,grok}-run.sh). Their shared inline-perl portable-timeout harness ended with `exit($? >> 8)`, which returns 0 when the spawned model CLI dies by a SIGNAL (OOM-killer SIGKILL, SIGSEGV crash, etc.) — Perl stores the signal in `$?`'s low 7 bits with 0 in the high byte. A crashed, deliverable-less attempt therefore printed `FARNSWORTH-*-DONE exit=0`, which the downstream provenance gate accepts as success, defeating the scripts' explicit fail-closed intent. Changed the final line to map signal deaths to the shell-conventional `128 + signum` (nonzero) while leaving normal exits, exec-fail 127, and the watchdog `exit 124` untouched.

Tradeoffs/limits: docs-only change to inline comments plus one perl line per script; no test or config files touched. Verified the new block with bash -n on all five scripts and behaviorally (clean=0, exit7=7, SIGKILL=137, SIGSEGV=139, exec-fail=127, timeout=124). I could not add a regression test without touching a *.test.* file (which the task asks to avoid), so the fix is covered by the manual verification above rather than an automated test.
