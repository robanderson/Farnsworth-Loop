# Code Tips (written by reviewer only; workers read-only)

## Invariants
- [2026-06-11, task-001] The worker briefing is EXACTLY `tips + "\n\nTASK: " + brief`. The `"\n\nTASK: "` separator is a fixed literal and is always present, even when `.code-tips.md` is absent (then the briefing begins with `\n\nTASK: `). Do not special-case the no-tips path to a bare `"TASK: "`.
- [2026-06-11, task-001] Feed the task brief contents verbatim into the briefing. Do not `.strip()`/`.rstrip()` the brief or the tips text before composing.
- [2026-06-11, task-001] Code in this repo must never write OR commit `.code-tips.md`; it is reviewer-authored and worker-read-only. Do not scaffold/commit an empty `.code-tips.md` from a worker task either.
- [2026-06-11, task-001] run.json field names are an exact contract: top-level `task_id, started_at, finished_at, base_commit, workers[]`; each worker `id, branch, worktree, exit_code, stdout_file, gate{passed, results[]{name, exit_code, autopsy}}`. `stdout_file` is the bare relative name `"w1.stdout"`; `worktree` is the relative string `"../<task-id>-w1"`.
- [2026-06-11, task-001] All gate commands run in order even if an earlier one fails (no short-circuit); `gate.passed` is the AND of all gate exit codes being 0.

## Gotchas
- [2026-06-11, task-001] Derive the repo root from `git rev-parse --show-toplevel`, not `os.getcwd()`/`Path.cwd()`. The worktree (`../<task-id>-w1`) and clean-tree check must be anchored to the git toplevel so the tool works when invoked from a subdirectory.
- [2026-06-11, task-001] Use timezone-aware UTC timestamps: `datetime.now(timezone.utc)`. Avoid `datetime.utcnow()` (deprecated 3.12+, returns naive) and avoid appending a literal `"Z"` to a naive isoformat.
- [2026-06-11, task-001] Never commit `__pycache__/`/`*.pyc`; add a `.gitignore`. Committed bytecode is a recurring hygiene failure and undercuts the `compileall` gate.

## Style
- [2026-06-11, task-001] CLI output is ASCII only — no `✓`/`✗` or other non-ASCII glyphs (terminal/encoding portability).
- [2026-06-11, task-001] Make the loop body callable in-process and testable: `run(...)` returns the run-log dict and accepts an injectable `cwd`; `cli.main(argv=None)` accepts an argv list. Tests should drive these directly rather than spawning `python3 -m farnsworth`.
- [2026-06-11, task-001] Prefer a small typed-error + thin-CLI split (raise a project error in the library, catch it in `cli.main` and print to stderr) over `sys.exit("msg")` for control flow inside library code.
